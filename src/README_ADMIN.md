# 电商管理系统使用说明

## 🎯 系统概述

这是一个完整的电商应用系统，包含用户前台和管理后台，数据通过Supabase持久化存储。

## 📱 功能模块

### 用户前台
- **商品浏览** - 浏览所有产品，支持搜索和排序
- **产品详情** - 查看产品详细信息
- **购物车** - 添加商品到购物车，管理数量
- **结账流程** - 填写收货信息、支付信息、确认订单
- **订单确认** - 查看订单确认信息

### 管理后台
通过菜单 → "管理后台" 进入

#### 产品管理
- 查看所有产品列表
- 删除产品
- 查看产品详情

#### 添加产品
支持添加以下信息：
- **产品名称*** (必填)
- **价格显示*** (必填) - 例如：¥10.00 每个
- **价格数值*** (必填) - 用于计算的数字
- **描述*** (必填)
- **农场/供应商*** (必填)
- **产地** (选填)
- **分类** (选填)
- **图片URL** (选填)
- **饮食标签** (选填) - VG(素食)、GF(无麸质)、DF(无乳制品)

#### 订单管理
- 查看所有订单
- 查看订单详情（客户信息、购买商品）
- 更新订单状态
  - 待处理
  - 处理中
  - 已发货
  - 已送达
  - 已取消

## 🚀 快速开始

### 1. 添加第一个产品

1. 打开菜单，点击 "管理后台"
2. 点击 "添加产品" 标签
3. 填写产品信息
4. 点击 "添加产品" 按钮

### 2. 管理产品

1. 在 "产品管理" 标签查看所有产品
2. 点击 "删除" 按钮可删除产品

### 3. 查看订单

1. 在 "订单管理" 标签查看所有订单
2. 使用下拉菜单更新订单状态

## 💡 使用提示

### 产品图片
- 可以使用任何公开的图片URL
- 建议使用高质量的产品图片
- 图片会自动调整大小以适应界面

### 价格格式
- **价格显示**：用于在界面上显示，可以包含单位（如 "¥10.00 每个"）
- **价格数值**：用于计算总价，只填数字（如 10.00）

### 产品分类
- 目前分类是文本字段
- 可以输入任何分类名称（如 "水果"、"蔬菜"、"零食" 等）
- 后续可以扩展为下拉选择

### 饮食标签
- **VG** - Vegan/Vegetarian 素食
- **GF** - Gluten Free 无麸质
- **DF** - Dairy Free 无乳制品

## 🔧 技术架构

### 前端
- React + TypeScript
- Tailwind CSS

### 后端
- Supabase Edge Functions (Hono框架)
- Supabase KV存储

### API端点
- `GET /products` - 获取所有产品
- `POST /products` - 创建产品
- `PUT /products/:id` - 更新产品
- `DELETE /products/:id` - 删除产品
- `GET /orders` - 获取所有订单
- `POST /orders` - 创建订单
- `PUT /orders/:id` - 更新订单状态

## 📝 数据结构

### 产品 (Product)
```json
{
  "id": "时间戳ID",
  "name": "产品名称",
  "price": "价格显示文本",
  "priceValue": 数字价格,
  "description": "产品描述",
  "location": "产地",
  "farm": "农场/供应商",
  "category": "分类",
  "images": ["图片URL数组"],
  "dietary": ["饮食标签数组"],
  "createdAt": "创建时间"
}
```

### 订单 (Order)
```json
{
  "id": "订单ID",
  "customerInfo": {
    "fullName": "客户姓名",
    "address": "地址",
    "city": "城市",
    "state": "州/省",
    "zipCode": "邮编",
    "country": "国家"
  },
  "items": [
    {
      "id": "产品ID",
      "name": "产品名称",
      "price": "价格显示",
      "priceValue": 数字价格,
      "quantity": 数量
    }
  ],
  "total": 订单总金额,
  "status": "订单状态",
  "createdAt": "创建时间"
}
```

## 🎨 自定义商店类型

这个系统非常灵活，可以轻松改造成售卖不同类型商品的商店：

### 电子产品商店
- 产品名称：iPhone 15 Pro
- 分类：智能手机
- 描述：最新款iPhone...

### 服装商店
- 产品名称：春季连衣裙
- 分类：女装/连衣裙
- 饮食标签可改为：尺码标签

### 图书商店
- 产品名称：《三体》
- 分类：科幻小说
- 农场/供应商 → 作者/出版社

### 美妆商店
- 产品名称：保湿面霜
- 分类：护肤品
- 描述：适合干性肌肤...

## ⚠️ 重要提示

1. **数据持久化**：所有数据存储在Supabase，刷新页面不会丢失
2. **订单自动保存**：用户完成购买后，订单会自动保存到数据库
3. **图片存储**：当前使用外部图片URL，如需上传功能可扩展Supabase Storage
4. **权限控制**：当前无认证系统，生产环境建议添加管理员登录功能
